1. Установить Jenkins
Требования:
- Jenkins должен быть контейнеризированный (Docker)
- Jenkins Home не должен перетираться после рестарта контейнера
- AD авторизация
- Jenkins должен быть доступен по 80 порту

2. Создать WEB-Portal
Требования:
- backend - Python (bottle)
- 2 сервера (с балансировкой)
- должен быть доступен, если 1 из backend - лежит
- должен быть доступен по 80 порту
- MySQL (MariaDB) - master-slave
- исходный код должен быть на GitHub
Что портал должен уметь:
- страничка авторизации (basic auth), роли должны сохраняться в базе
- запретить неавторизированные запросы
- на страничке показывать данные из базы (произвольная табличка с данными) 
- возможность обновлять и добавлять данные через UI
- весь портал должен быть оформлен при помощи Twitter Bootstrap

3. Подготовить Jenkins задачу для автоматической заливки портала (Groovy pipeline)
- входящие параметры для Jenkins задачи - имя ветки (from GitHub)
- должно заливать портал на 2 сервера без даунтайма (rolling update)
